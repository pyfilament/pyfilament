Welcome to Pyfilament — a lightweight Python task framework that turns any async function into a managed, durable task with a single `@task` decorator. Add retries, timeouts, caching, rate limiting, concurrency controls, and distributed workers to your async code without restructuring it.

## Get up and running in three steps

<Steps>
  <Step title="Install Pyfilament">
    Add `pyfilament` to your project using `uv` or `pip`.

    ```bash
    uv add pyfilament
    ```
  </Step>
  <Step title="Decorate your function with @task">
    Import `task` from `filament` and apply it to any async function.

    ```python
    from filament import task

    @task(tries=3, timeout=10.0)
    async def fetch_data(url: str) -> dict:
        ...
    ```
  </Step>
  <Step title="Run it">
    Call the function normally. Pyfilament manages execution, retries, and state tracking automatically.

    ```python
    from asyncio import run

    run(fetch_data("https://api.example.com/data"))
    ```
  </Step>
</Steps>

## Feature highlights

<CardGroup cols={2}>
  <Card title="Retries & Backoff" icon="rotate">
    Automatically retry failing tasks with configurable attempt counts, fixed delays, and exponential backoff. Control which exception types trigger a retry.
  </Card>
  <Card title="Timeouts & Caching" icon="clock">
    Cancel tasks that exceed a time limit and cache results by input arguments — with optional TTL — so repeated calls skip redundant work.
  </Card>
  <Card title="Rate Limiting" icon="gauge">
    Enforce per-task rate limits (calls per second) and maximum concurrency caps to protect downstream services and shared resources.
  </Card>
  <Card title="Distributed Workers" icon="server">
    Dispatch tasks to remote workers via Redis-backed queues. Scale horizontally and monitor every run through the built-in observability UI.
  </Card>
</CardGroup>
